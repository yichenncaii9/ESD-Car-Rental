**Firestore initialization** (immediately after config, top-level):
Use the try/except pattern from atomic/report_service — set `db = None` on failure. Print `[activity_log] Firestore init failed: {e}` on error. This allows the worker to run even without Firebase credentials (degraded mode — skips Firestore write but still POSTs to websocket_server).

**connect_with_retry(max_attempts=5) function:**
Loop from attempt 1 to max_attempts. Try `pika.BlockingConnection(pika.ConnectionParameters(host=RABBITMQ_HOST, port=RABBITMQ_PORT, heartbeat=60))`. On success, print `[activity_log] Connected to RabbitMQ on attempt {attempt}` and return connection. On exception, compute `wait = 2 ** attempt` (2, 4, 8, 16, 32 seconds). Print `[activity_log] RabbitMQ not ready (attempt {attempt}/{max_attempts}): {e}. Retrying in {wait}s`. If attempt == max_attempts, re-raise. Else `time.sleep(wait)`.

**callback(ch, method, properties, body) function:**
Wrap entire body in try/except/finally where finally always calls `ch.basic_ack(delivery_tag=method.delivery_tag)`.

Inside try:
1. Parse: `data = json.loads(body)`, extract `report_id = data.get("report_id")`, `severity = data.get("severity")`.
2. Firestore write (if db is not None):
   ```python
   db.collection("activity_log").add({
       "report_id": report_id,
       "event":     "report.new",
       "timestamp": datetime.datetime.utcnow().isoformat(),
       "severity":  severity,
   })
   print(f"[activity_log] Firestore write: report_id={report_id}")
   ```
   Wrap this in a nested try/except that prints `[activity_log] Firestore write failed: {e}` on error — do NOT re-raise.
3. POST to websocket_server:
   ```python
   try:
       requests.post(
           f"{WEBSOCKET_URL}/notify",
           json={"report_id": report_id, "event": "activity_logged",
                 "severity": severity, "message": "Activity logged"},
           timeout=5
       )
   except Exception as e:
       print(f"[activity_log] websocket notify failed: {e}")
   ```
4. Outer except: `print(f"[activity_log] callback error: {e}")`

**main() function:**
```python
def main():
    connection = connect_with_retry()
    channel = connection.channel()
    channel.exchange_declare(exchange=EXCHANGE_NAME, exchange_type="topic", durable=True)
    channel.queue_declare(queue=QUEUE_NAME, durable=True)
    channel.queue_bind(queue=QUEUE_NAME, exchange=EXCHANGE_NAME, routing_key=ROUTING_KEY)
    channel.basic_qos(prefetch_count=1)
    channel.basic_consume(queue=QUEUE_NAME, on_message_callback=callback)
    print(f"[activity_log] Waiting for messages on {QUEUE_NAME}...")
    channel.start_consuming()

if __name__ == "__main__":
    main()
```

CRITICAL: `exchange_type="topic"` — NOT "direct" or "fanout". Must match the declaration in `composite/report_issue/app.py` exactly or pika will raise ChannelClosedByBroker.

Do NOT add a Flask HTTP server — this is a pure AMQP consumer (no exposed port, no app.run).
