The frontend image has VITE_* variables baked at build time by scripts/build-images.sh — do NOT add any env vars to the Deployment. No secrets, no volumes, no runtime ConfigMap references.

k8s/frontend/deployment.yaml:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: frontend
  namespace: default
spec:
  replicas: 1
  selector:
    matchLabels:
      app: frontend
  template:
    metadata:
      labels:
        app: frontend
    spec:
      containers:
        - name: frontend
          image: esd-frontend:latest
          imagePullPolicy: Never
          ports:
            - containerPort: 8080
          livenessProbe:
            httpGet:
              path: /
              port: 8080
            initialDelaySeconds: 15
            periodSeconds: 20
            failureThreshold: 5
          readinessProbe:
            httpGet:
              path: /
              port: 8080
            initialDelaySeconds: 15
            periodSeconds: 20
            failureThreshold: 5
```

k8s/frontend/service.yaml — NodePort so the Vue.js app is reachable from Docker Desktop host:
```yaml
apiVersion: v1
kind: Service
metadata:
  name: frontend
  namespace: default
spec:
  selector:
    app: frontend
  type: NodePort
  ports:
    - port: 8080
      targetPort: 8080
      nodePort: 30080
```

Apply all manifests in dependency order, then run the smoke test. Execute these commands in sequence:

```bash
# 1. Apply shared ConfigMap first (other manifests reference common-config)
kubectl apply -f k8s/shared/

# 2. Apply infrastructure
kubectl apply -f k8s/rabbitmq/
kubectl apply -f k8s/kong/

# 3. Apply atomic services (parallel-safe — no inter-dependency)
kubectl apply -f k8s/vehicle-service/
kubectl apply -f k8s/booking-service/
kubectl apply -f k8s/driver-service/
kubectl apply -f k8s/report-service/
kubectl apply -f k8s/pricing-service/

# 4. Apply wrappers + websocket-server
kubectl apply -f k8s/openai-wrapper/
kubectl apply -f k8s/googlemaps-wrapper/
kubectl apply -f k8s/stripe-wrapper/
kubectl apply -f k8s/twilio-wrapper-http/
kubectl apply -f k8s/websocket-server/

# 5. Apply composite services
kubectl apply -f k8s/composite-book-car/
kubectl apply -f k8s/composite-cancel-booking/
kubectl apply -f k8s/composite-report-issue/
kubectl apply -f k8s/composite-resolve-issue/

# 6. Apply workers
kubectl apply -f k8s/twilio-worker/
kubectl apply -f k8s/activity-log/

# 7. Apply frontend
kubectl apply -f k8s/frontend/

# 8. Wait for pods to initialize (workers may CrashLoopBackOff initially — expected)
kubectl get pods --watch &
sleep 90
kill %1 2>/dev/null || true

# 9. Open port-forward for Kong in background (smoke test uses localhost:8000)
kubectl port-forward svc/kong 8000:8000 8001:8001 &
PF_PID=$!
sleep 5

# 10. Run smoke test
bash scripts/verify_phase6.sh

# 11. Clean up port-forward
kill $PF_PID 2>/dev/null || true
```

If Kong returns 502 for all routes: check `kubectl logs deployment/kong` for config parse errors in the ConfigMap.
