<objective>
Create manifests for the 4 composite services and 2 AMQP worker Deployments. Workers get no Service (nothing calls into them — they consume from RabbitMQ and call out).

Purpose: Composite services orchestrate the three user-facing scenarios (K8S-07). Workers are the async processors. Both layers depend on atomic services (Plan 03) and wrappers (Plan 04) being DNS-resolvable first.

CRITICAL — Inter-service URL env vars: Composite Flask apps use os.environ.get("VEHICLE_SERVICE_HOST", "vehicle_service:5001") etc. The defaults contain Docker Compose underscore names which do NOT resolve in K8s DNS. Every composite ConfigMap MUST set the hyphenated K8s Service name for each atomic/wrapper host env var. Failing to do so causes all composite-to-atomic HTTP calls to fail with DNS NXDOMAIN.

Output: 15 manifest files covering 6 services
</objective>

Workers are AMQP consumers only — no HTTP ports, no Kubernetes Service needed. They self-heal via pika retry logic when RabbitMQ is initially unavailable.

NOTE: Workers have NO Service manifest by design. They consume from RabbitMQ queues and call outbound services — nothing calls into them over HTTP. A K8s Service would be unused and misleading. This is the intentional exception to K8S-01's Deployment+Service+ConfigMap pattern for HTTP-serving pods.
