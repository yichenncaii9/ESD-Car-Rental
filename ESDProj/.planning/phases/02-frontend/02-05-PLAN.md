Firebase JWT facts (from RESEARCH.md):
- Algorithm: RS256
- iss claim format: https://securetoken.google.com/{FIREBASE_PROJECT_ID}  (exact — trailing slash matters)
- Public key endpoint: https://www.googleapis.com/robot/v1/metadata/x509/securetoken@system.gserviceaccount.com
  Returns: { "kid1": "-----BEGIN CERTIFICATE-----\n...\n-----END CERTIFICATE-----\n", "kid2": "..." }
- Kong limitation: Standard JWT plugin does NOT auto-fetch JWKS. Key is embedded statically in kong.yml.
  Firebase rotates keys every ~7 days. Acceptable for university project.
- key_claim_name: iss — Kong matches the token's iss claim against jwt_secrets[].key

Kong JWT plugin config pattern:
```yaml
consumers:
  - username: firebase-frontend
    jwt_secrets:
      - algorithm: RS256
        key: "https://securetoken.google.com/{FIREBASE_PROJECT_ID}"
        rsa_public_key: |
          -----BEGIN CERTIFICATE-----
          [PEM certificate from x509 endpoint]
          -----END CERTIFICATE-----
        secret: "not-used-for-rs256"

plugins:  # per-route, inside each route's plugins array
  - name: jwt
    config:
      key_claim_name: iss
      claims_to_verify:
        - exp
```

```bash

```

This returns a JSON object like:
```json
{
  "abc123": "-----BEGIN CERTIFICATE-----\nMIID...==\n-----END CERTIFICATE-----\n",
  "def456": "-----BEGIN CERTIFICATE-----\nMIIF...==\n-----END CERTIFICATE-----\n"
}
```

To find which key your Firebase tokens are currently using:
1. Log in to your Firebase-powered app (or use the Firebase Console to generate a test token)
2. Decode the token at https://jwt.io — look at the Header section for the `kid` field
3. Use the PEM certificate whose key ID (the JSON key like "abc123") matches your token's `kid`

Copy the matching PEM certificate value (the full "-----BEGIN CERTIFICATE-----...-----END CERTIFICATE-----" string).

Also confirm your FIREBASE_PROJECT_ID from Firebase Console -> Project Settings -> General.
